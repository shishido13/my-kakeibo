# 機能追加計画：Dashboard フィルタリング機能

## 1. 概要
ダッシュボード画面において、取引履歴を条件（期間、カテゴリ、キーワード）で絞り込める機能を追加します。
これにより、特定の支出を素早く特定したり、期間ごとの集計を視覚的に確認しやすくしたりします。

## 2. 実装方針

### 2.1. バックエンド
以下のクエリパラメータをサポートするよう、APIとCRUD処理を拡張します。
- `start_date`: 開始日 (YYYY-MM-DD)
- `end_date`: 終了日 (YYYY-MM-DD)
- `category_id`: カテゴリID
- `payer`: 支払者名（前方一致または完全一致） [新規追加]
- `keyword`: 店舗名、内容、説明に含まれるキーワード

### 2.2. フロントエンド
#### Pinia ストア (`stores/transaction.ts`)
- `fetchTransactions` アクションを拡張し、`payer` を含むフィルタ条件を受け取れるようにします。

#### Dashboard ビュー (`views/DashboardView.vue`)
- フィルタ検索セクションに以下の入力を追加します：
    - **支払者選択**: 支払者マスタから選択するプルダウン、または自由入力。

- **「検索」ボタン**: クリック時にストアの `fetchTransactions` を再実行します。
- **「リセット」ボタン**: フィルタ条件を初期化し、全件表示に戻します。

## 4. 実装ステップ
1.  **バックエンドの修正**: 
    - `backend/app/api/v1/transactions.py` の `read_transactions` に `payer` パラメータを追加。
    - `backend/app/crud/transaction.py` の `get_transactions` に `payer` によるフィルタリングロジック（完全一致/部分一致）を追加。
2.  **ストアの修正**: `fetchTransactions` アクションで `payer` パラメータを扱えるよう拡張。
3.  **UI コンポーネントの追加**: `DashboardView.vue` に `payer` を含む入力フォームを追加。
4.  **ロジックの接続**: フォームの状態とストアのアクションを連携。
5.  **動作確認**: 全ての条件（特に新規追加の支払者）で正しく絞り込めるか確認。

## 5. 考慮事項
- **初期表示**: デフォルトでは今月のデータを表示するか、全件表示するかを検討（まずは全件表示から開始）。
- **ユーザー体験**: 入力するたびに検索する（デバウンス処理）か、検索ボタンを押した時に実行するか。初回はシンプルに「検索ボタン」方式を採用。
