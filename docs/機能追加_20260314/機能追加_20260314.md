## 1. プロジェクト概要

* **名称:** My Kakeibo
* **フェーズ:** Phase 2 (PDFデータ化 & 一括登録機能の実装)
* **コア価値:** 月末にスキャンしたPDF（レシート・支出カード）をAIで解析し、確認・修正を経てシームレスにDB登録する。

## 2. 追加・更新技術スタック

* **AI API:** Gemini 3 Flash (2026年最新のコストパフォーマンス・モデル)
    * **理由:** 1.5 Flashに比べ、複雑なレシートや独自カードの構造化出力精度が15%以上向上。
    * **料金:** 100万トークンあたり約$0.10（1回の解析で1円未満）。
* **Frontend 追加要素:** * PDFプレビュー機能 (PDF.js 等)
* インライン編集可能な確認用データテーブル


* **Backend 追加要素:** * `python-multipart` (ファイルアップロード用)
* `google-generativeai` (Gemini SDK)

## 3. 追加・修正データモデル (Schema)

### 3.1. `source_type` の厳密化

`Transactions` テーブルの `source_type` フィールドに以下のリテラルを想定。

* `manual`: 手入力
* `ai_pdf`: PDF解析経由
* `csv_import`: CSVインポート経由

## 4. API 設計 (新規追加)

### 4.1. `POST /api/v1/receipts/analyze`

* **概要:** PDFを解析し、登録候補（JSON）を返す。
* **Request:** `file: UploadFile` (PDF)
* **Internal Process (`ai_service.py`):**
1. Gemini 1.5 Flash にPDFを送信。
2. System Instruction で「日付、金額、店舗、商品、決済者(payer)、カテゴリ」を抽出するよう指示。
3. `Categories` および `Payers` のマスタデータ名をコンテキストとして与え、AIに既存マスタへのマッピングを試みさせる。


* **Response:** `List[TransactionCreate]` (DB登録前の候補データリスト)

## 5. UI/UX 設計 (一括登録フロー)

### 5.1. 「データ化確認」ビューの構成

1. **PDF Preview (Left Pane):** アップロードしたPDFを表示し、内容を目視確認できるようにする。
2. **Candidate Table (Right Pane):**
* AIが抽出したデータをテーブル形式で表示。
* 各セルはクリックで `input` に切り替わり、即座に修正可能。
* 行削除ボタン。
「PDFアップロードから登録完了まで」のユーザー体験と、コンポーネント間の責務を定義します。

### 5.1. 画面遷移フロー

1. **一覧画面**: `Header` に配置された「PDF一括登録」ボタンをクリック。
2. **ファイル選択**: ファイルダイアログでPDFを選択。
3. **解析中**: ローディング画面を表示（Gemini 3 Flash が解析中）。
4. **ImportVerifyView**: 解析完了後、この画面へ遷移。
5. **完了**: 確定ボタン押下後、一覧画面へ戻り、トースト通知を表示。

### 5.2. `ImportVerifyView.vue` のコンポーネント構成

この画面は「突き合わせ」の精度を最大化するため、以下の **2カラム・スプリットレイアウト** を採用します。

#### **A. 左ペイン: PDF Viewer**

* **役割**: スキャンした原本の視認。
* **機能**:
* ページ送り機能。
* ズーム機能（小さなレシートの数字を確認するため）。
* `Intersection Observer` 等を利用し、右側のテーブルで選択中の行に対応する「日付」付近へ自動スクロールする補助機能（将来拡張）。



#### **B. 右ペイン: Interactive Data Table**

AIが抽出した JSON 配列を `v-for` で展開し、**「全セルが即編集可能」**な状態で表示します。

| 項目 | 入力形式 | 仕様・こだわり |
| --- | --- | --- |
| **日付** | `input type="date"` | カレンダー選択、または直接入力。 |
| **金額** | `input type="number"` | 右寄せ。カンマ表示はフォーカス外れた際に行う。 |
| **店舗** | `input type="text"` | AIの抽出結果を保持。 |
| **内容** | `input type="text"` | 商品名やサービス名。 |
| **カテゴリ** | `select` | `Categories` マスタから選択。 |
| **決済者** | `select` | `Payers` マスタから選択（AIが推論した初期値）。 |
| **操作** | `button` | 行削除ボタン（ゴミデータ排除用）。 |

### 5.3. インライン編集のキーボードショートカット (DX向上)

エンジニアとしての入力効率を重視し、以下の挙動を実装します。

* **Tab / Shift+Tab**: セル間の横移動。
* **Enter**: 下の行の同じフィールドへ移動（Excelライクな挙動）。
* **一括適用ボタン**: テーブルヘッダーに「一括選択」機能を設け、チェックした行の `payer` や `category` を一括で書き換える。

## 5.4. 解析ステート管理 (Frontend)

`stores/useImportStore.ts` で以下の状態を管理します。

```typescript
interface ImportState {
  originalFile: File | null;      // プレビュー用PDFファイル
  pendingTransactions: Transaction[]; // AIから返ってきた修正中の配列
  isAnalyzing: boolean;           // ローディング状態
}

```

3. **Action Bar:**
* `決済者を一括変更`: 選択した行、または全ての行の `payer` を一括で書き換える。
* `DBへ登録`: 修正後のデータを一括で `POST /api/v1/transactions/bulk` へ送信。
